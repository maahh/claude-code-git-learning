このアプリについて

GitHubアカウントに接続することで、実際のコマンドを実行しながら学習できます。クイズではなく、本物のリポジトリを使った実践型の学習です。
必要なもの

• GitHubアカウント（無料）
• GitHub Personal Access Token（PAT）

PATの取得方法: github.com → Settings → Developer settings → Personal access tokens → Tokens (classic)

必要な権限: repo（リポジトリの読み書き）にチェック
セキュリティについて

PATはこのブラウザのlocalStorageのみに保存されます。外部サーバーには一切送信されません。GitHub公式API（api.github.com）にのみ使用します。
Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → repo にチェック

Lv.1 — STEP 1 / 5
STEP 1 / 5
リポジトリとは何か
GitHubが「プロジェクトの変更履歴を記録する箱」であることを体感する
解説

Gitリポジトリとは、プロジェクトのすべての変更履歴を記録する「箱」です。普通のフォルダと違い、いつ・誰が・何を変えたかをすべて記憶しています。

GitHubは、そのリポジトリをクラウド上に保存・共有するサービスです。Claude Codeはリポジトリの内容を読み取り、変更し、コミット（保存）します。Worktreeなど高度な機能もリポジトリがあって初めて使えます。

Claude Codeでの意味

Claude Codeを起動すると、まず現在フォルダのリポジトリ構造を読み取ります。git logで変更履歴も把握した上で作業を開始します。リポジトリがないとWorktree（並行作業）が使えず、AIを最大限活用できません。

GitHubで確認する

アプリが作成した練習用リポジトリをGitHub APIで取得して、リポジトリの実データを表示します。

Lv.1 — STEP 2 / 5
STEP 2 / 5
git clone でローカルに複製する
GitHubのリポジトリをMacに持ってきて、Claude Codeが作業できる状態にする

git cloneは、GitHubにあるリポジトリをローカル（あなたのMac）にコピーするコマンドです。クローンしたフォルダの中には .git フォルダが自動で作られ、すぐにGitが使える状態になります。

Claude Codeはクローンしたフォルダの中で起動することで、ファイルの読み書き・コミット・ブランチ操作など、すべての作業ができるようになります。

実践：ターミナルで実行してみよう

以下のコマンドをターミナル（またはClaude Codeのチャット）で実行してください

1
練習用リポジトリをクローンする
Terminal 
# デスクトップにクローン（場所はどこでもOK）
cd ~/Desktop
git clone https://github.com/あなたのユーザー名/claude-code-git-practice.git
cd claude-code-git-practice
💡 クローンURLはGitHubリポジトリページの「Code」ボタン → 「HTTPS」からコピーできます
2
クローンできたか確認する
Terminal 
# ファイル一覧を確認
ls -la
# .git フォルダがあればOK！

# Gitの状態確認
git status

GitHubで完了を確認する

クローン自体はローカル操作なので直接確認できませんが、リポジトリの現在の状態をGitHub APIで表示します。

Lv.1 — STEP 3 / 5
STEP 3 / 5
git add / commit / push
変更をGitHubに保存して、実際のコミット履歴をAPIで確認する

Gitでの保存は3段階です。add（ステージング）→ commit（確定）→ push（クラウドへ送信）。pushするとGitHubのリポジトリに反映され、どこからでも確認できます。

Claude Codeに変更を頼んだ後も、このサイクルを意識することで「何が変わったか」を正確に把握できます。

実践：ファイルを作ってGitHubに送ってみよう

以下のコマンドをクローンしたフォルダ内で実行してください

1
練習ファイルを作る
Terminal
# クローンしたフォルダにいることを確認
pwd

# 練習ファイルを作成
echo "# はじめてのコミット" > hello.md
echo "Claude CodeでGitを学習中！" >> hello.md
2
add → commit → push する
Terminal
# 変更をステージングする
git add hello.md

# 変更をコミットする（メッセージは日本語でOK）
git commit -m "はじめてのコミット: hello.mdを追加"

# GitHubに送信する
git push origin main
💡 pushが完了したらボタンを押してください。GitHubで実際のコミットを確認します。

GitHubでコミットを確認する

pushが完了したら確認ボタンを押してください。hello.mdのコミットがGitHub APIで検出されると完了です。

Lv.1 — STEP 4 / 5
STEP 4 / 5
.gitignore でAPIキーを守る
機密情報をGitHubに公開しないための必須設定

.gitignore ファイルに書いたファイル名・パターンは、Gitの管理対象外になります。APIキーや.envファイルが誤ってGitHubに公開される事故を防ぐために必須の設定です。

Claude Codeを使っていると、AIが設定ファイルを生成することがあります。pushする前に必ず機密情報が含まれていないか確認し、.gitignoreで保護しましょう。

実践：.gitignore を作ってpushしよう

APIキーが書かれた.envを守る設定を実際に行います

1
.envファイル（機密情報の例）を作る
Terminal
# 実際のAPIキーは使わず、テスト用の内容で作成
echo "ANTHROPIC_API_KEY=sk-ant-テスト用-絶対にGitHubに上げてはいけない" > .env
⚠️ このあとgitignoreで必ず除外します
2
.gitignore を作成して .env を除外する
Terminal
# .gitignore を作成（.envを除外）
cat > .gitignore << 'EOF'
.env
.env.local
*.log
.DS_Store
node_modules/
EOF

# .envが追跡されていないことを確認（表示されなければOK）
git status
3
.gitignore をコミット・push する
Terminal
git add .gitignore
git commit -m "セキュリティ: .gitignoreを追加（.env等を除外）"
git push origin main

GitHubで確認する

pushが完了したら確認してください。GitHubに.gitignoreが存在し、.envが存在しないことをAPIで確認します。

リポジトリファイル一覧（GitHub APIより）
Claude Codeに貼れるプロンプト
コピーしてClaude Codeに貼り付ける
このプロジェクト用の.gitignoreを作成してください。 必ず除外するもの： - .env（APIキー・機密情報） - node_modules/ - *.log - .DS_Store 作成後、git statusで.envが追跡されていないことを確認してください。

STEP 5 / 5
git log をリアルタイムで確認する
これまでの作業履歴をGitHub APIで取得して、コマンドの結果と見比べる

git logは、コミットの歴史を時系列で表示するコマンドです。「いつ・誰が・何をしたか」がわかります。Claude Codeが変更を加えた後に git log --oneline で確認する習慣をつけると、AIの作業内容を正確に把握できます。
