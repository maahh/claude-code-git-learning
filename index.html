STEP 1 / 5
リポジトリとは何か
Claude Codeが認識する「プロジェクトの箱」の概念
解説

Gitリポジトリとは、プロジェクトのすべての変更履歴を記録する「箱」のようなものです。普通のフォルダと違うのは、いつ・誰が・何を変えたかをすべて記憶している点です。

たとえば「昨日の状態に戻したい」「この変更は誰がいつ追加したのか」という疑問に答えられるのが、リポジトリの仕組みです。

フォルダの中に .git という隠しフォルダが作られると、そのフォルダはリポジトリになります。これがGitの「記憶装置」です。

Claude Codeでの意味

Claude Codeを起動すると、まず現在のフォルダのリポジトリ構造を読み取ります。どんなファイルがあり、どんな変更履歴があるかを把握した上で、AIが作業を開始します。

リポジトリがない状態でClaude Codeを使うと、Worktree（並行作業）機能が使えません。後で詳しく学びますが、Worktreeが使えると作業効率が格段に上がります。

STEP 2 / 5
git init
Claude Codeが動く場所を作る
解説

git init はフォルダをGitリポジトリに変えるコマンドです。たった一行、実行するだけで完了します。

このコマンドを実行すると、フォルダの中に .git という隠しフォルダが作られます。これ以降、そのフォルダでの変更がすべて記録されるようになります。

Claude Codeは「プロジェクトを始めてください」という指示を受けたとき、自動で git init を実行してくれることもありますが、自分で実行できると何が起きているかが分かります。

コマンド
Terminal 
# まず、作業したいフォルダに移動する
cd ~/Desktop/my-project

# Gitリポジトリとして初期化する
git init

# 結果: "Initialized empty Git repository" と表示されれば成功
Claude Codeでの意味

Worktree（並行作業）機能は、gitリポジトリが必須です。

Worktreeとは、同じプロジェクトの異なるブランチを、別のフォルダで同時に作業できる機能です。Claude Codeはこれを使って「AIに複数の作業を同時にやらせる」ことができます。

git init がすべての始まりです。

STEP 3 / 5
git add / git commit
作業をスナップショット保存して「元に戻せる状態」を作る
解説

Gitでの保存は2段階です。git add（ステージング）と git commit（確定保存）です。

写真撮影に例えると、git add は「カメラに構える」、git commit は「シャッターを押す」イメージです。commitするたびに、その瞬間のプロジェクト全体のスナップショットが保存されます。

コミットが積み重なることで、「3つ前の状態に戻したい」という操作が可能になります。

コマンド
Terminal 
# 変更したすべてのファイルをステージングする
git add .

# または特定のファイルだけ
git add index.html

# 変更を確定保存（メッセージは日本語でもOK）
git commit -m "初回コミット: プロジェクト作成"

# 確認: コミット履歴を見る
git log --oneline
Claude Codeでの意味

AIに大きな変更を頼む前に、必ずコミットしましょう。

Claude Codeに「このファイルを全部書き直して」と頼んで失敗しても、事前にコミットがあれば git checkout . で元に戻せます。

Claude Code自身も、作業の節目に自動でコミットしてくれることがあります。コミットメッセージを日本語で書くよう指示することもできます。

STEP 4 / 5
.gitignore
Claude Codeに見せてはいけないファイルを隠す（APIキー等）
解説

.gitignore ファイルに書いたファイル名やパターンは、Gitの管理対象外になります。「なかったこと」として扱われます。

特に重要なのはセキュリティです。APIキーやパスワードが書かれた .env ファイルを誤ってGitHubに公開してしまう事故が後を絶ちません。.gitignore はその予防策です。

Claude Codeを使う場合は、Claude Codeが生成した設定ファイルに機密情報が入っていないかを必ず確認し、.gitignore で保護しましょう。

設定例（.gitignoreの中身）
.gitignore 
# APIキー・環境変数ファイル（絶対に公開しない）
.env
.env.local

# ログファイル（容量が大きく不要）
*.log

# パッケージフォルダ（サイズが巨大）
node_modules/

# macOSが自動生成するファイル（不要）
.DS_Store

# IDEの設定（チームで共有不要なもの）
.vscode/settings.json
セキュリティ注意

Claude Codeでコードを書く際、GitHubにpushする前に必ずセキュリティチェックを行いましょう。

確認項目: .env やAPIキー・トークン類が含まれていないか / CLAUDE.md や設定ファイルに機密情報が混入していないか / .gitignore に機密ファイルが適切に登録されているか。

STEP 5 / 5
git status / git log
現在の状態と変更履歴を確認する
解説

git status は「今どんな変更が起きているか」を確認するコマンドです。どのファイルが変更されたか、ステージングされているか、コミット待ちか、が一目でわかります。

git log は過去のコミット履歴を表示します。「いつ・何を変えたか」の記録が見られます。--oneline オプションを付けると、一行ずつコンパクトに表示されます。

この2つのコマンドを使いこなすと、Claude Codeが何をしたか、何が変わったかを正確に把握できます。

コマンド
Terminal 
# 現在の変更状態を確認（よく使う！）
git status

# コミット履歴を一行で見る（わかりやすい）
git log --oneline

# 直近3件だけ表示する
git log --oneline -3

# 特定のコミットの変更内容を見る（IDを指定）
git show abc1234
Claude Codeでの意味

AIが変更した内容を「確認→コミット」のサイクルで管理するのが安全なClaude Code活用の基本です。

Claude Codeに指示を出す → git status で変更を確認 → 問題なければ git commit でスナップショット保存、というサイクルを習慣にすると、AIの作業ミスに気づきやすくなり、いつでも巻き戻しができる状態を保てます。
